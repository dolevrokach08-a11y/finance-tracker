## הפרומפט (להדביק כלשונו)

---

אתה מעצב מחדש אפליקציית ווב עברית (RTL) לניהול פיננסי אישי, לקראת פתיחתה למשתמשים חדשים.

**על המוצר:** כלי מעקב תיק השקעות שמרכז הכובד שלו הוא אנליטיקת תשואות מדויקת (TWR/MWR, השוואה למדדים). קהל היעד: משקיעים ישראלים עצמאיים, חובבי דאטה אבל לא בהכרח אנשי פיננסים. מיקוד המוצר: **צד ההשקעות תחילה** — דף הבית, התיק והאנליטיקות הם הבמה המרכזית; מעקב הוצאות/משכנתא/מס הם משניים.

**מה לשמר (זהות קיימת):**
- שפת "טרמינל פיננסי" כהה בדפי ההשקעות — מקצועית ומבודלת; אל תהפוך אותה לפסטל גנרי.
- ירוק = רווח, אדום = הפסד, בעקביות.
- רצועת הטיקר החיה בראש דף התיק.
- עברית RTL כשפה ראשית; מספרים ומטבעות תמיד בכיוון LTR מבודד.

**אילוצי מימוש (חשוב!):** האפליקציה היא vanilla HTML/CSS/JS בלי framework ובלי build. העיצוב צריך להיות בר-יישום כ-CSS tokens + מבני HTML פשוטים. אל תציע קומפוננטות React או ספריות UI. עדיפות ל-CSS variables, flex/grid.

**המשימות, לפי סדר עדיפות:**

1. **דף הבית — שני מצבים:** (א) משתמש חדש בלי נתונים: empty-state מזמין עם צעדים ראשונים ("הוסף מזומן → הוסף החזקות → שחזר היסטוריה"), בלי גרפים מזויפים ובלי מקפים ריקים; (ב) משתמש עם נתונים: היררכיה ברורה — שווי תיק ותשואה כמספרי-גיבור, השאר משני.
2. **כרטיס התשואה בתיק:** היום מוצגים 4 מספרים (TWR, Base, Live, MWR) בלי הסבר. עצב כרטיס עם מספר-גיבור אחד (TWR), תווית משנה קצרה, אייקון "?" שפותח הסבר, ושורת הקשר שמסבירה למה התשואה באחוזים יכולה להיות ירוקה כשהרווח בשקלים אדום.
3. **ניווט:** צמצום 10 טאבי התיק ל-5 קטגוריות (סיכום / התיק שלי / ניתוח / כלים / הגדרות), ורצועת ניווט עליונה אחידה בין חמשת דפי האפליקציה (היום כל דף שונה).
4. **אחידות ערכת נושא:** גרסה כהה + בהירה לכל האפליקציה מתוך אותם tokens (היום דף אחד בהיר והשאר כהים).
5. **מובייל 390px:** גרסאות מובייל לדף הבית ולסיכום התיק — הטאבים, הכרטיסים והטבלאות.

**רשימת הליקויים המלאה מהסקירה שביצענו מצורפת (design-spec-phase4a.md) — התייחס לפחות לכל סעיפי P0 ו-P1.**

**פורמט התוצר המבוקש:**
1. לוח design tokens: פלטה (בהיר+כהה), סולם טיפוגרפי, ריווח, רדיוסים — בפורמט CSS variables מוכן להדבקה.
2. מוקאפים למסכים: דף בית (ריק + מלא), סיכום תיק, טאב החזקות, מובייל — ייצוא כ-HTML עצמאי (סטנדאלון) כדי שנוכל ליישם ישירות.
3. הערת רציונל קצרה לכל החלטה מרכזית.
